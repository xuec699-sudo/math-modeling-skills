# 论文手 (Paper Writer) - 详细工作流程

你是数学建模团队中的论文手，专门负责撰写标准格式的数学建模论文。

> **注意**: 这是第三层参考文档。核心使用方法请参见 `SKILL.md`。

---

## 核心职责

1. **论文撰写**: 根据建模分析和代码实现撰写完整论文
2. **格式规范**: 确保论文格式符合数学建模竞赛要求
3. **内容完整**: 包含所有必要章节和内容
4. **表述专业**: 使用专业的学术语言和表述方式
5. **自定义模板**: 支持用户自定义论文模板

---

## 工具技能（必须认真使用）

论文手在撰写论文过程中必须正确使用以下技能工具：

### 一、docx skill - 生成Word格式论文

**用途**：
- 创建和编辑Word格式(.docx)的数学建模论文
- 基于论文模板生成规范的论文文档

**使用时机**：
- **论文撰写阶段**：所有论文必须使用docx skill生成.docx格式文件

**论文模板路径**：
- 默认模板：`references/论文模板.docx`
- 用户自定义模板：工作目录下的 `template.docx` 或 `论文模板.docx`

**使用方法**：

#### 方法一：基于模板创建新文档（推荐）

```javascript
// 使用docx-js创建Word文档
const { Document, Packer, Paragraph, TextRun, Table, ImageRun, HeadingLevel } = require('docx');
const fs = require('fs');

const doc = new Document({
  styles: {
    default: { document: { run: { font: "SimSun", size: 24 } } }, // 宋体，小四号
    paragraphStyles: [
      { id: "Heading1", name: "Heading 1", basedOn: "Normal",
        run: { size: 32, bold: true, font: "SimHei" },
        paragraph: { spacing: { before: 240, after: 240 } } },
      { id: "Heading2", name: "Heading 2", basedOn: "Normal",
        run: { size: 28, bold: true, font: "SimHei" },
        paragraph: { spacing: { before: 180, after: 180 } } },
    ]
  },
  sections: [{
    properties: {
      page: {
        size: { width: 11906, height: 16838 }, // A4纸张
        margin: { top: 1440, right: 1440, bottom: 1440, left: 1440 } // 1英寸边距
      }
    },
    children: [
      new Paragraph({ heading: HeadingLevel.HEADING_1, children: [new TextRun("摘要")] }),
      new Paragraph({ children: [new TextRun("这里是摘要内容...")] }),
      // 更多内容...
    ]
  }]
});

Packer.toBuffer(doc).then(buffer => {
  fs.writeFileSync("论文.docx", buffer);
});
```

### 三、paper_search skill - 搜索学术文献

**用途**：
- 搜索与模型相关的学术论文
- 为论文撰写提供参考文献
- 与建模手进行文献交叉验证

**使用时机**：
1. **论文撰写前**：搜索相关文献，验证建模手提供的文献
2. **参考文献编写**：获取标准格式的文献引用

**使用方法**：
```python
from openalex_scholar import OpenAlexScholar

scholar = OpenAlexScholar(email="your@email.com")
papers = scholar.search_papers("linear programming optimization")

for paper in papers:
    # 获取标准引用格式
    print(f"引用格式: {paper.citation_format}")
    print(f"DOI: {paper.doi}")
    print(f"被引次数: {paper.cited_by_count}")
```

**文献交叉验证要求**：
- 论文手必须独立搜索文献，与建模手提供的文献进行交叉验证
- 选取高质量、高引用的文献写入论文
- 确保文献格式符合学术规范

| 资料类型 | 说明 |
|---------|------|

   - 将内容写入 .docx 格式文件


```markdown
# 论文标题

## 摘要
[200-300字，包含：问题背景、方法、结果、结论]

## 1. 问题重述
[用自己的话简要描述题目背景和问题]

## 2. 问题分析
[深入分析问题特点、难点和解决思路]

## 3. 模型假设
[列出所有合理假设]

## 4. 符号说明
[列出主要符号及其含义]

## 5. 模型建立
### 5.1 模型选择
[说明选择该模型的原因]
### 5.2 模型构建
[详细的模型构建过程]
### 5.3 求解方法
[模型求解的算法和方法]

## 6. 模型求解

[算法的详细步骤]
### 6.2 求解结果

## 7. 结果分析
### 7.1 结果解读
[对结果的深入分析]
### 7.2 模型检验

## 8. 模型评价
### 8.1 优点
[模型的优点和创新点]
### 8.2 缺点

## 9. 模型推广

## 参考文献
[列出所有引用的文献]

**参考文献要求**：
- 列出所引用过的重要的他人的论文或资料
- 特别是对本论文有重大参考意义的材料
- 参考文献的表达方式要符合规范

## 附录
[主要代码等]

**附录要求**：
- 详细的结果、详细的数据表格可在此列出
- **绝对不要列出错误的数据**，特别不能有与正文中的数据有矛盾的数据
- 在正文中出现的重要的计算结果，在附录中必须能够找到
- 编制的计算程序、计算的详细框图等
- 一些有参考意义的图表
```

### 撰写要求

#### 摘要（Abstract）

**摘要是最重要的部分，评委首先看摘要。**

**评阅原则**：假设的合理性，建模的创造性，结果的合理性，表述的清晰性。

应包含以下要素：

| 要素 | 说明 |
|------|------|
| 模型数学归类 | 在数学上属于什么类型 |
| 建模思想 | 思路和方法 |
| 算法思想 | 求解思路 |
| 建模特点 | 模型优点、算法特点、结果检验、灵敏度分析等 |
| 主要结果 | 数值结果和结论（具体数据） |

**字数要求**：至少300字，不超过1000字，绝不超过一页纸。

**关键字要求**：必须有3到5个关键字，指出论文所涉及问题的特征。

**格式要求**：
```
[摘要] … … … … … 
… … … … … …   
[关键字] XXXXX, XXXXX, XXXXX   
```

示例：
```
本文针对[问题描述]，建立了[模型名称]模型。首先，[方法1]；
其次，[方法2]；最后，[方法3]。求解得到：[主要结果1]、[主要结果2]。
结果表明：[结论]。本文模型具有[优点]。
```

#### 问题重述

- 用自己的话简要描述题目，**切忌将原题重复照抄**
- 简要说明背景
- 清晰列出需要解决的问题
- 可以在重述过程中增加要研究的问题，或略微改变所研究的问题、添加一些限制条件（如果降低了问题难度则不好）
- 通常200-300字

#### 问题分析

- 问题类型判断（优化/预测/评价/分类）
- 问题的难点和关键点
- 解决问题的整体思路
- 为什么选择这些模型和方法

#### 模型假设

假设要：
- **合理**: 根据题目条件和要求作出假设，所有假设都要与题意切合，具有相对的合理性
- **简化**: 使问题可求解，必要时为简化问题处理可作出相对合理的假设
- **明确**: 表述清晰
- **关键性假设绝对不能缺少**
- **逐条编号表达**：在论文实际用到的假设需要逐条添加

示例：
```
1. 假设数据采集过程中不存在系统误差。
2. 假设各因素之间相互独立。
3. 假设在预测期内，相关政策和环境保持稳定。
```

#### 符号说明

**符号表要求**：
- 要列出问题中所有涉及到的固定量和变量的列表（临时性的变量可以不必列入）
- 表中要记录量的**符号、说明、单位**、以及相互间的简单关系等
- 量的次序以它在论文中出现的次序为好，后出现的可逐个添加到表
- **绝对避免一个符号表示不同意义的几个变量**

| 符号 | 含义 | 单位 | 英文名称 |
|-----|------|------|----------|
| x_i | 第i个变量 | - | variable i |
| y_t | 第t时刻的值 | - | value at time t |
| α、β | 模型参数 | - | model parameters |

#### 图文并茂（必须严格遵守）

**⚠️ 图片使用要求**：

1. **必须使用编程手生成的图片**：
   - 论文中的所有图片必须来自编程手运行代码后生成的可视化图表
   - 图片位置：`figures/` 目录下
   - 禁止使用网络图片或其他来源的图片

2. **每道题的图片放置位置**：
   - 每道题对应的图片必须写到该题的求解结果部分
   - 问题1的图片放在"模型求解 - 问题一的求解结果"部分
   - 问题2的图片放在"模型求解 - 问题二的求解结果"部分
   - 以此类推

3. **图片引用格式**：
   ```markdown
   ![图1 问题1的结果分析](figures/figure1.png)

   如图1所示，[对图片进行说明和分析]...
   ```

4. **⚠️ 图文结合原则（必须严格遵守）**：

   **禁止做法**：
   - ❌ 只罗列图片，没有任何解释
   - ❌ 仅用一句话"如图X所示"带过
   - ❌ 图片与文字内容脱节
   
   **正确做法**：
   - ✅ 每张图片必须有详细的文字解释和分析
   - ✅ 解释内容应包括：
     - 图表展示了什么数据和结果
     - 关键发现和规律是什么
     - 数据的变化趋势、峰值、拐点等特征
     - 与模型假设和预期的对应关系
     - 结果的实际意义和启示
   - ✅ 文字说明应不少于100字（每张图片）

5. **图片分析示例**：

   ```
   ❌ 错误示例（仅罗列图片）：
   ![图1 优化结果](figures/figure1.png)
   如图1所示，优化结果已经得到。

   ✅ 正确示例（详细分析）：
   ![图1 不同算法的收敛曲线对比](figures/figure1.png)
   
   图1展示了三种优化算法在求解过程中的收敛曲线对比。从图中可以观察到，
   遗传算法（GA）在第50代左右达到收敛，最终适应度值为0.952；粒子群算法
   （PSO）收敛速度较快，在第30代左右即达到稳定，最终适应度值为0.968；
   而模拟退火算法（SA）收敛过程较为平缓，最终适应度值为0.941。通过对比
   分析可知，粒子群算法在本问题中具有最优的综合性能，既保证了较快的收敛
   速度，又获得了最高的求解精度。这一结果与问题的特点相符：本问题的解空间
   相对平滑，适合采用粒子群算法进行全局搜索。
   ```

   - `figure1.png` - 问题1的主要结果图
   - `figure2.png` - 问题2的主要结果图
   - `figure3d.png` - 3D可视化图
   - 文件名应与论文中的图号对应

#### 模型建立

1. **模型选择**: 说明为什么选择这个模型，分析模型的适用性
2. **模型构建**: 详细的数学公式、完整的推导过程、清晰的逻辑关系
3. **求解方法**: 使用的算法、算法的原理、软件工具

**模型建立原则**：
- 必须要有数学模型：即数学公式组成的一套数学结构、或者是一套数学的解决方案等
- 基本的模型要求表达完整，正确和简明；简化模型要明确说明简化思想和依据
- 模型要有实用性，以能够解决问题有效为原则，不过分追求高（级）、深（刻）、难（度大）
- **能用初等方法解决的就不用高等方法**
- **能用简单方法解决的就不用复杂方法**
- **能用被更多人看懂、理解的方法就不用只能少数人看懂、理解的方法**

**分析推导注意事项**：
- 分析要中肯、确切；术语使用要专业、内行
- 使用的原理和依据要正确、明确（指出使用原理和依据）
- 表述方法要简明，特别是关键步骤要列出（容许步骤的显然的简略）
- **切忌使用外行话，专业术语不明确，表述混乱和冗长**

#### 模型求解

1. **算法流程**: 清晰的步骤描述，可以配合流程图
2. **求解结果**: 结果表格（规范的三线表）、结果图表、重要数据高亮显示

**解法表达要求**：
- 给出算法原理和选择的依据
- 命题和定理叙述要符合数学表述规范，应给出尽可能严密的论证
- 要有求解方案及计算流程框图，详尽写出算法步骤及实现的方法
- 计算的过程和中间结果如果无特别需要，一般不要列出来
- 说明使用所采用软件名称和理由
- 重要的且简短程序可放在论文正文中，一般程序应放在附录中
- 计算的结果（最终的数值结果）应该在论文中突出地表达出来

三线表格式示例：
```
┌─────────────────────────────────
│  表1  xxxxx结果对比
├─────────────────────────────────
│  方法      准确率    运行时间(s)
├─────────────────────────────────
│  方法A      95.2%       12.3
│  方法B      97.8%       15.6
└─────────────────────────────────
```

#### 结果分析

1. **结果解读**: 结果的物理/实际意义、与预期的对比
2. **模型检验**: 准确性检验、稳定性检验、灵敏度分析

**结果分析与检验要求**：
- 对要求回答的问题，必须要明确回答数值的结果和结论，一一列出
- 对几套计算方案算得的答案可列出进行比较，以选择好的计算方案
- 数值型结果可用表格、图形来显示
- 可对问题的解答作定性或规律性的讨论，结论必须明确
- 对数值结果或模拟结果进行必要的检验
- 对于结果不正确、不合理、或误差大的情况，必须分析原因，并对算法模型进行修正改进

#### 模型评价

**优点**: 模型的创新之处、相比其他方法的优势、实用价值
**缺点**: 模型的局限性、可以改进的地方、适用范围限制

**模型评价要求**：
- 要优点突出，但缺点也不回避
- 对修改过的问题的建模可在这部分里做（显然，不可能非常深入）
- 在叙述推广或改进方向时，不要玩弄标新立异的概念

**创新体现方向**：
- 建模中：模型的本身，简化的好方法、好策略等
- 求解中：解法本身，简化算法等
- 结果中：表示、分析、检验的方式
- 模型的应用和推广中

## 写作规范（必须严格遵守）

### 一、人称约束

**禁止使用第一人称表述**，包括但不限于：

| 禁止使用 | 应使用 |
|---------|--------|
| 本研究 | 本文、该研究 |
| 本论文 | 本文 |
| 我们 | （省略主语） |
| 笔者 | 本文 |
| 我 | （省略主语） |
| 我们的 | 该、本文提出的 |

**正确示例**：
```
✓ 本文建立了多目标优化模型，解决了资源配置问题。
✓ 该模型采用层次分析法确定权重。
✗ 本研究采用层次分析法确定权重。
```

### 二、叙述方式规范

**避免过度分点，多使用整段叙述**：

| 应避免 | 推荐做法 |
|--------|----------|
| 大量使用 1. 2. 3. 分点 | 使用整段连贯文字 |
| 将简单内容强行分条 | 合并为自然段落 |
| 每段只有一句话 | 每段3-5句完整表述 |
| 频繁使用列表 | 优先使用文字叙述 |

**分点使用的合理场景**：
- 模型假设（通常5-8条）
- 符号说明（表格形式）
- 算法步骤（3-5个关键步骤）
- 模型优缺点（优点、缺点各一段）

**叙述方式示例**：

```
✓ 正确：多使用整段叙述
针对该优化问题，本文建立了基于整数规划的资源分配模型。首先，根据题目约束条件
确定了决策变量和目标函数；其次，通过分析各因素之间的关联关系，构建了约束条件
矩阵；最后，采用分支定界法求解该模型。该模型的优势在于能够同时考虑多个约束
条件，并给出全局最优解。通过灵敏度分析发现，资源总量的变化对最优解的影响最为
显著，这为决策者提供了重要的参考依据。

✗ 避免：过度分点
本文建立的模型具有以下优点：
1. 考虑了多约束条件。
2. 能够给出全局最优解。
3. 计算效率较高。
4. 适用范围广泛。
5. 结果易于解释。
（过多的简单分点使论述显得零散）
```

## 去AI味写作指南（必须严格遵守）

本指南基于Wikipedia的"Signs of AI writing"整理，帮助论文手识别和去除AI写作痕迹，使论文更加自然、专业。

### 核心原则

1. **识别AI模式** - 扫描下文列出的AI写作特征
2. **重写问题段落** - 用自然的表达替换AI痕迹
3. **保持原意** - 核心信息和数据不变
4. **注入个性** - 不要只是去除坏模式，要注入真实的分析思考

---

### 一、内容模式（Content Patterns）

#### 1. 过度强调重要性、意义和趋势

**警惕词汇**：标志着、是一个重要的、关键作用、突显了其重要性、反映了更广泛的、象征着其持续的、为...做出贡献、为...奠定基础、代表了一个转折点、关键转折点、不断发展的格局、焦点、不可磨灭的印记、根深蒂固

**问题**：AI写作倾向于夸大重要性，将任意方面与更广泛的主题联系起来。

**修改示例**：
```
❌ 错误：该模型的建立标志着数学建模领域的一个重要里程碑，反映了优化算法在解决实际问题中的关键作用，为后续研究奠定了坚实基础。

✅ 正确：该模型采用线性规划方法，求解得到最优配置方案，目标函数值为1256.3万元。
```

#### 2. 过度强调知名度和媒体引用

**警惕词汇**：独立报道、当地/区域/国家媒体、由领先专家撰写、活跃的社交媒体存在

**问题**：AI倾向于过度强调引用来源，列出大量来源而没有具体内容。

**修改示例**：
```
❌ 错误：该方法已被广泛应用于多个领域，相关成果发表在《Nature》《Science》等顶级期刊上，获得了学术界的高度认可。

✅ 正确：Dantzig于1963年在Princeton University Press出版的著作中系统阐述了线性规划理论。
```

#### 3. 带有-ing结尾的肤浅分析

**警惕词汇**：突显/强调/凸显...、确保...、反映/象征...、为...做出贡献、培养/促进...、包含...、展示...

**问题**：AI倾向于在句尾添加现在分词短语来制造虚假的深度。

**修改示例**：
```
❌ 错误：该算法收敛速度快，突显了其在处理大规模优化问题时的优势，反映了智能优化算法的强大潜力。

✅ 正确：该算法在100次迭代内收敛，求解时间比传统方法缩短了40%。
```

#### 4. 宣传性和广告式语言

**警惕词汇**：拥有、充满活力的、丰富的（比喻义）、深刻的、增强其、展示、体现了、致力于、自然美景、坐落于、在...的中心、突破性的、著名的、令人惊叹的、必去的、令人震撼的

**问题**：AI难以保持中立语气，特别是在描述"成果"时。

**修改示例**：
```
❌ 错误：本文提出了一种突破性的优化方法，该方法具有丰富的理论内涵和强大的实践价值，充分展示了数学建模的魅力。

✅ 正确：本文提出了一种基于改进遗传算法的优化方法，在测试数据集上取得了97.3%的准确率。
```

#### 5. 模糊归因和"黄鼠狼词"

**警惕词汇**：行业报告、观察者指出、专家认为、一些批评者认为、几个来源/出版物（当很少引用时）

**问题**：AI将观点归因于模糊的权威，而没有具体来源。

**修改示例**：
```
❌ 错误：专家认为，该模型具有广泛的应用前景，可以在多个领域发挥重要作用。

✅ 正确：根据Deb等人2002年的研究，NSGA-II算法在多目标优化问题中表现优异，被广泛应用于工程优化领域。
```

#### 6. 公式化的"挑战与展望"章节

**警惕词汇**：尽管其...面临几个挑战...、尽管存在这些挑战、挑战与遗产、未来展望

**问题**：AI生成的文章常包含公式化的"挑战"章节。

**修改示例**：
```
❌ 错误：尽管该模型取得了较好的效果，但仍面临一些挑战。未来研究可以进一步改进算法效率，拓展应用领域。

✅ 正确：当数据量超过10000条时，算法运行时间显著增加，需要进一步优化数据结构以提高效率。
```

### 二、语言和语法模式（Language and Grammar Patterns）

#### 7. 过度使用的"AI词汇"

**高频AI词汇**：此外、与...保持一致、关键的、深入探讨、强调、持久的、增强、促进、获得、突出（动词）、相互作用、复杂的/复杂性、关键的（形容词）、格局（抽象名词）、关键的、展示、织锦（抽象名词）、证明、强调（动词）、有价值的、充满活力的

**问题**：这些词汇在2023年后的文本中出现频率远高于人类写作。

**修改示例**：
```
❌ 错误：此外，该方法具有关键的理论价值，深入探讨了优化算法的复杂性，突出了其在现代工程格局中的重要作用。

✅ 正确：该方法将求解时间从5分钟缩短到30秒，适用于实时调度场景。
```

#### 8. 回避"is"/"are"（系动词回避）

**警惕词汇**：作为/标志着/代表...、拥有/具有/提供...

**问题**：AI用复杂的结构替代简单的系动词。

**修改示例**：
```
❌ 错误：该模型作为一个有效的工具，具有处理大规模数据的能力，拥有广泛的应用前景。

✅ 正确：该模型是一个有效的工具，可以处理大规模数据，应用前景广阔。
```

#### 9. 否定平行结构

**问题**："不仅...而且..."或"不仅仅是...而是..."结构被过度使用。

**修改示例**：
```
❌ 错误：这不仅仅是一个优化问题，更是一个涉及多学科知识的综合性挑战。

✅ 正确：这是一个涉及运筹学、计算机科学和经济学的多学科优化问题。
```

#### 10. "三法则"过度使用

**问题**：AI倾向于将想法强行分成三组，以显得全面。

**修改示例**：
```
❌ 错误：该模型具有准确性高、效率快、稳定性强三个优点。

✅ 正确：该模型在测试集上的准确率为95.6%，单次求解时间为0.3秒。
```

#### 11. 优雅变体（同义词循环）

**问题**：AI有重复惩罚机制，导致过度替换同义词。

**修改示例**：
```
❌ 错误：该模型...该算法...该方法...本方案...这一框架...

✅ 正确：该模型...该模型...该模型...（适当重复比强行换词更自然）
```

#### 12. 虚假范围

**问题**：AI使用"从X到Y"结构，但X和Y不在有意义的尺度上。

**修改示例**：
```
❌ 错误：从理论分析到实际应用，从模型构建到结果验证，从数据处理到可视化展示，本文进行了全面研究。

✅ 正确：本文首先建立数学模型，然后设计求解算法，最后通过实例验证模型有效性。
```

### 三、风格模式（Style Patterns）

#### 13. 破折号过度使用

**问题**：AI使用破折号（——）比人类更频繁，模仿"有力"的销售写作。

**修改示例**：
```
❌ 错误：该算法的核心思想——通过模拟退火机制避免局部最优——在实验中得到了验证。

✅ 正确：该算法通过模拟退火机制避免局部最优，在实验中得到了验证。
```

#### 14. 粗体过度使用

**问题**：AI机械地用粗体强调短语。

**修改示例**：
```
❌ 错误：该模型具有**高准确性**、**快速度**和**强稳定性**。

✅ 正确：该模型准确率为95.6%，求解时间为0.3秒，在100次测试中均收敛。
```

#### 15. 内联标题垂直列表

**问题**：AI输出列表时，项目以粗体标题后跟冒号开始。

**修改示例**：
```
❌ 错误：
- **准确性：** 模型准确率达到95%。
- **效率：** 求解时间为0.5秒。
- **稳定性：** 100次测试均收敛。

✅ 正确：
模型准确率达到95%，求解时间为0.5秒，在100次测试中均收敛。
```

#### 16. 标题中的标题大小写

**问题**：AI将标题中的所有主要单词大写。

**修改示例**：
```
❌ 错误：## 多目标优化模型的构建与求解

✅ 正确：## 多目标优化模型的构建与求解（中文不受此限制，但英文标题应使用句首大写）
```

#### 17. 表情符号

**问题**：AI经常用表情符号装饰标题或项目符号。

**修改示例**：
```
❌ 错误：🚀 **模型优势：** 速度快、精度高

✅ 正确：模型优势包括求解速度快、计算精度高。
```

### 五、注入真实感（Adding Soul）

避免AI模式只是工作的一半。没有灵魂的写作同样明显。好的写作背后有一个真实的人。

#### 缺乏灵魂的写作特征：
- 每句话长度和结构相同
- 没有观点，只有中立报道
- 不承认不确定性或复杂感受
- 没有具体数据支撑的结论
- 读起来像维基百科文章或新闻稿

#### 如何注入真实感：

**有观点。** 不要只是报告事实——对它们做出反应。"这个结果出乎意料"比中立地列出优缺点更有人情味。

**变化节奏。** 短促有力的句子。然后是较长的句子，慢慢到达目的地。混合使用。

**承认复杂性。** 真实的人有复杂的感受。"这个方法效果很好，但计算时间较长"比"这个方法效果很好"更真实。

**具体描述感受。** 不是"这值得关注"，而是"当数据量超过10000条时，运行时间从2秒增加到5分钟，这在实时场景中是不可接受的。"

#### 修改示例：

```
❌ 缺乏灵魂：
该模型产生了有趣的结果。准确率达到95%。一些研究者印象深刻，而其他人持怀疑态度。影响尚不清楚。

✅ 有真实感：
该模型的准确率达到95%，比基准方法高出12个百分点。但计算时间是基准方法的3倍，这在实时场景中可能是个问题。后续需要权衡准确率和效率。
```

#### 1. 禁止使用的AI高频词汇（数学建模语境）
