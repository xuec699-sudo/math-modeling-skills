> **来源**: KyrieZhang329/MathModeling-skills problem-classifier + 本技能定制

## 二、分类决策树

```
问题要求什么类型的输出？
│
├─ 数值排名/得分/等级 → 评价 (类型1)
│  └─ 指标独立？→ TOPSIS/熵权法
│  └─ 指标相关？→ 层次分析+模糊综合评价
│
├─ 未来数值/趋势 → 预测 (类型2)
│  └─ 数据充分(≥30点)？→ ARIMA/时间序列
│  └─ 数据稀疏？→ 灰色预测 GM(1,1)
│
├─ 最优方案/配置 → 优化 (类型3)
│  └─ 线性关系？→ 线性规划/整数规划
│  └─ 非线性？→ 非线性规划/智能优化
│
├─ 物理规律/动态过程 → 机理/物理 (类型4)
│  └─ 干涉/衍射？→ 光学干涉模型 (2.7A)
│  └─ 热/扩散？→ 热传导模型 (2.7B)
│  └─ 波动/振动？→ 波动方程模型 (2.7C)
│  └─ 一般动态？→ 微分方程模型 (2.3)
│
├─ 类别/标签 → 分类 (类型5)
│  └─ 线性可分？→ 逻辑回归/SVM
│  └─ 非线性？→ 随机森林/神经网络
│  └─ 无标签？→ 聚类 (K-means/DBSCAN)
│
├─ 路径/流量/连接 → 图论 (类型6)
│  └─ 最短路径？→ Dijkstra/Floyd
│  └─ 最大流？→ Ford-Fulkerson
│  └─ 匹配？→ 匈牙利算法
│
└─ 变量关系/影响 → 统计/回归 (类型7)
   └─ 单因变量？→ 线性/非线性回归
   └─ 多因变量？→ 多元回归/结构方程
   └─ 分布特征？→ 假设检验/方差分析
```

## 三、子问题分解原则

1. **按输出拆**: 每个子问题产生不同类型输出 → 不同类型
2. **按变量拆**: 不同子问题涉及不同变量集 → 可能需要不同类型
3. **按阶段拆**: 前一子问题的输出是后一子问题的输入 → 建模依赖链

## 四、常见误分类陷阱

| 陷阱 | 示例 | 纠正 |
|------|------|------|
| 评价当预测 | "预测各方案的优劣" | 实际是评价：已有指标数据，只需排序 |
| 优化当评价 | "选出最优方案" | 可能有连续决策变量，需建立优化模型 |
| 统计当机理 | 用回归拟合物理数据 | 如果有物理定律，应建立机理模型而非纯统计 |
| 分类当聚类 | "将样本分为3类" | 如果类别含义已知→分类；未知→聚类 |
| 单一类型套全局 | 全题标为"优化问题" | 按子问题拆分，可能 Q1评价 Q2优化 Q3预测 |

## 五、分类产物格式

对每个子问题输出：

```json
{
  "subquestion": "Q1",
  "primary_type": "机理/物理",
  "primary_subtype": "光学干涉",
  "secondary_type": null,
  "required_output": "外延层厚度 d (nm)",
  "classification_reason": "题目要求基于光谱干涉条纹计算薄膜厚度，核心是物理机理建模",
  "candidate_method_families": ["Fabry-Perot干涉模型", "双光束干涉模型"],
  "unsuitable_methods": ["纯统计回归（无物理基础）", "深度学习（数据不足）"],
  "data_dependency": "Q1结果供Q2使用",
  "risk_flags": ["折射率色散可能影响精度", "多光束干涉效应可能需修正"]
}
```

## 六、与算法选择映射

分类完成后，使用 `references/problem-algorithm-map.json` 匹配具体算法：

| 问题类型 | 首选算法 | 备选算法 |
|---------|---------|---------|
| 评价 | TOPSIS, 层次分析 | 熵权法, 模糊综合评价 |
| 预测 | ARIMA, 灰色预测 | LSTM, Prophet |
| 优化 | 线性规划, 整数规划 | 遗传算法, 粒子群 |
| 机理 | 微分方程, 物理公式推导 | 数值仿真 |
| 分类 | 逻辑回归, SVM | 随机森林, XGBoost |
| 图论 | Dijkstra, Floyd | Ford-Fulkerson, 匈牙利 |
| 统计 | OLS回归, 假设检验 | 岭回归, LASSO |
