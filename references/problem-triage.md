# 竞赛选题六维评估框架

> **来源**: math-modeling-competition-workflow
> **适用时机**: 竞赛开始阶段，需在多道题中选择时使用。

---

## 一、六维评估维度

| 维度 | 评估内容 |
|------|----------|
| **1. 数学类型** | 优化/预测/分类/仿真/微分方程/图论/统计/混合 |
| **2. 数据可得性与清洗风险** | 数据是否完整？缺失率？是否需要大量清洗？ |
| **3. 算法复杂度与编码量** | 需要多少代码？是否有现成库？预计编码时间？ |
| **4. 评价清晰度** | 是否有客观评价指标？还是依赖主观论证？ |
| **5. 论文写作难度** | 需要多少图/表？证明负担？可解释性要求？ |
| **6. 团队匹配度** | 领域知识、编程能力、时间约束是否匹配？ |

## 二、推荐逻辑

总分 = 维度1×0.15 + 维度2×0.20 + 维度3×0.15 + 维度4×0.20 + 维度5×0.15 + 维度6×0.15

一票否决：数据维度=1 或 团队匹配度=1

## 三、选题报告模板

对每道题输出：

| 维度 | 评分(1-5) | 说明 |
|------|-----------|------|
| 数学类型 | — | 主要类型+混合类型 |
| 数据可得性 | — | 数据情况+清洗风险 |
| 算法复杂度 | — | 核心算法+预计编码量 |
| 评价清晰度 | — | 评价指标+主观/客观 |
| 写作难度 | — | 图表数+证明量+可解释性 |
| 团队匹配度 | — | 知识匹配+编程匹配+时间 |
| **加权总分** | **—** | |

## 四、典型题目类型快速判断

| 题目特征 | 可能的类型 | 建议方法储备 |
|----------|-----------|-------------|
| 分配/调度/路径规划 | 优化(LP/IP/NLP) | Gurobi, 遗传算法, 蚁群算法 |
| 根据历史数据预测未来 | 预测(时序/回归) | ARIMA, LSTM, XGBoost, Prophet |
| 分类/识别/诊断 | 分类 | 逻辑回归, SVM, 随机森林, CNN |
| 物理过程/传播/变化 | 微分方程 | ODE/PDE, SIR, 有限元, 数值解 |
| 评价/排序/打分 | 评价/决策 | AHP, TOPSIS, 熵权法, 模糊综合评价 |
| 网络/连接/关系 | 图论/网络 | 最短路径, 最大流, 社区检测 |
| 生成/合成/模拟 | 仿真 | 蒙特卡洛, 元胞自动机, 智能体仿真 |
