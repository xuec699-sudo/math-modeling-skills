# 数学建模竞赛论文写作规范（基于 Nature Writing 方法论）

本文档将 Nature 期刊科学写作方法论适配到数学建模竞赛论文。
核心原则：先建论证，再写句子。从证据出发组织段落。

---

## 每条子问题的标准写作结构

每个子问题（4.x）必须包含三个完整小节：

### 4.x.1 模型建立（Model Building）

段落序列（必须逐段覆盖）：

**段落1 — 问题定位**
- 一句话说清这个子问题要解决什么
- 为什么现有方法/简单方法不够

**段落2 — 建模思路**
- 选择什么数学工具/方法
- 为什么选这个方法（与问题的匹配性论证）

**段落3 — 数学推导**
- 写出完整的数学模型：变量定义 → 目标函数 → 约束条件
- 公式必须编号，变量必须解释物理/数学含义
- 不要说"我们用线性回归"，而要写：
  "设 y_i 为观测量，x_i 为特征向量，假定 y_i = β·x_i + ε_i，
   ε_i ~ N(0,σ²)。参数 β 通过最小化残差平方和估计：
   min_β Σ(y_i - β·x_i)²"

**段落4 — 方法优势**
- 这个方法为什么比简单替代方案好
- 它的局限/假设是什么（诚实声明）

### 4.x.2 模型求解与结果分析（Solving & Results）

段落序列：

**段落1 — 求解过程**
- 模型如何求解（解析解/数值解/算法）
- 关键参数是如何选择的（调参依据）

**段落2 — 核心结果**
- 最重要的数字结果 + 含义解释
- 结果是否符合预期？为什么？

**段落3 — 交叉验证/鲁棒性**
- 泛化性能指标（CV-R², MAE, RMSE 等）
- 指标说明了什么

**段落4 — 物理/工程解释**
- 结果对原问题的物理意义是什么
- 与工程实际的一致性

### 4.x.3 图表说明

- 每张图/表插入后，紧接一段文字说明
- 说明必须包含：①图展示了什么 ②读者应看到什么关键信息 ③这个信息意味着什么

---

## 摘要写作：挑战-贡献结构

```
field context → bottleneck → approach overview → 
  Q1 result (key number) → Q2 result → ... → 
    overall significance → boundary
```

每条子问题的摘要句：做了什么 + 用了什么方法 + 关键数字 + 数字的含义

---

## 段落流（Paragraph Flow）检查清单

写完后逐段检查：

1. 这一段有且只有一个核心信息吗？
2. 首句宣布了这一段的要旨吗？
3. 每个后续句子与上一句有明确的逻辑关系吗（因果/对比/递进/例证）？
4. 术语在使用前定义了吗？
5. 有没有哪句可以移到另一段？

如果任何一项检查未通过，拆分/重组段落。

---

## 结果叙事：证据阶梯

```
问题动机 → 模型公式 → 求解方法 → 关键数字 → 
  交叉验证 → 对比/消融 → 物理机理 → 工程意义
```

不要跳级。读者需要看到完整链条才能理解模型的价值。

---

## 动词精度

| 证据强度 | 推荐动词 |
|----------|----------|
| 强（充分验证） | 证明、确认、实现 |
| 中（统计显著） | 表明、揭示、发现 |
| 弱（趋势/推测） | 暗示、可能、初步显示 |

不要用"证明"描述仅通过统计推断的结果。

---

## 具体禁止项

1. 禁止"黑箱跳跃"：从问题直接跳到结果，中间没有模型推导过程
2. 禁止"裸数字"：给出数字不解释含义
3. 禁止"图表孤儿"：插入图表后没有解释文字
4. 禁止"方法堆砌"：只列方法名不解释为什么选它
5. 禁止"空洞结论"：说"模型效果好"不说"好在哪、什么指标、意味着什么"
