# 数学建模竞赛论文写作规范 v4.0

> **适用**：所有论文生成阶段。Agent 在 `paper_write` 和 `content_assembly` 阶段**必须**遵循本规范。
> **来源**：综合 KyrieZhang329/MathModeling-skills 论文写作规则 + G5.1/G5.2 质量门禁

---

## 一、基本规则

### 从产物写，不从记忆写

论文每个段落必须有对应的上游产物支撑：

| 论文部分 | 对应产物 |
|---------|---------|
| 问题重述 | `problem_parse.json` |
| 问题分析 | `problem_classification.json` + `question_dependency.md` |
| 模型假设 | `model_assumptions.md` |
| 符号说明 | `global_symbol_table.md` |
| 数据预处理 | `data_report.md` |
| 模型构建 | `final_method_explanation.md` |
| 模型求解 | 代码 + `run_summary.json` |
| 结果分析 | `final_result_analysis.md` |
| 鲁棒性分析 | `robustness_report.md` |
| 图表 | `figure_table_plan.md` + 图源文件 |

### 禁止编造

永远不要编造：数据、数值结果、参考文献、图表、参数值、实验结果、模型性能声明。

## 二、G5.1 字数下限（按节）

不同节有不同最低实质性字数。达不到的节计为 `status: under_floor`，拒绝进入下一阶段。

| 论文节 | 最低中文字数 | 最低英文单词 | 理由 |
|--------|------------|------------|------|
| 摘要 | 200 | 150 | 需覆盖：问题+方法+关键结果+鲁棒性 |
| 问题重述 | 300 | 220 | 背景+总目标+子问题+约束 |
| 问题分析 | 400 | 280 | 分解逻辑+依赖映射+数据探索 |
| 模型假设 | 200 | 150 | 每条假设注明必要性和影响范围 |
| 符号说明 | 150 | 100 | 含单位、类型；跨子问题统一 |
| 数据预处理 | 300 | 220 | 来源+清洗操作+剩余风险 |
| **模型构建（每子问题）** | **600** | **450** | **这是最长的分节：假设+符号+数学公式+建模步骤+评估指标** |
| 模型求解（每子问题） | 250 | 180 | 求解器+计算流程+计算量说明 |
| 结果分析（每子问题） | 500 | 380 | 数值结果+解释+基线对比+图表引用 |
| 方法选择说明（每子问题） | 300 | 220 | 淘汰的方法+选择理由 |
| 鲁棒性与敏感性分析 | 350 | 250 | 稳定结论+脆弱结论+扰动细节 |
| 模型优缺点 | 300 | 220 | ≥2优点+≥2缺点，每个有证据支持 |
| 结论 | 300 | 220 | 逐子问题直接回答+关键数值+局限 |

**总字数目标：15,000-22,000字，18-22页**

### 字数不足时的处理

如果某节低于下限，writer 必须：
1. 报告 `status: under_floor`
2. 标注缺失的子内容（哪些子节缺少实质性内容）
3. **禁止用填充性文字凑数** — 填充文字会触发 paper-polisher 拒绝

## 三、G5.2 三维讨论规则

**论文中每个数值结果必须从以下5个维度中选择至少3个进行讨论**：

### 五个讨论维度

| # | 维度 | 定义 | 证据来源 |
|---|------|------|---------|
| 1 | **敏感性/鲁棒性** | 参数/权重/输入 ±10% 扰动下，该数值如何变化？ | `robustness_report.md` |
| 2 | **物理/领域含义** | 该数值在真实世界中代表什么？偏大还是偏小？合理还是意外？ | 领域知识+文献 |
| 3 | **基线对比** | 该数值与基线方法的结果相比如何？提升/下降了多少？ | `final_result_analysis.md` 中的对比表 |
| 4 | **跨子问题一致性** | 该数值与其它子问题的相关数值是否一致？（如 Q2 总需求 ≈ Q3 总分配量） | `cross_question_consistency.md` |
| 5 | **不确定性/置信区间** | 置信范围是多少？标准误差？Bootstrap CI？ | `final_result_analysis.md` |

### 违规示例

```
FAIL: "RMSE = 2.4" — 裸数值，无任何讨论
FAIL: "预测精度达到 95%" — 无基线对比，无敏感性分析
PASS: "RMSE = 2.4，与基线方法（RMSE=3.1）相比降低22.6%（维度3）；
      参数λ在±20%范围内扰动时RMSE波动<5%，结论稳定（维度1）；
      该精度满足竞赛要求的±0.5误差界限（维度2）"
```

### 最低要求

- 每个数值结果：≥3个维度
- 优先选择证据最充分的维度
- 如果某维度确实没有证据，标注为 `待验证` 而非编造

## 四、论文编写风格

### 推荐

- 清晰的技术性写作
- 短段落（3-5句）
- 方法-结果显式链接
- 克制的结论
- 一致的符号记法
- 每张图/表在正文中有引用和解释

### 避免

- 空洞赞美模型
- 模糊声明（如"效果良好"）
- 过多形容词
- 无证据的因果断言
- 装饰性措辞
- AI典型的填充语（"值得注意的是""总的来说""此外"过度使用）
- 不在摘要中过度承诺

## 五、声明检查清单

写任何声明前，逐项确认：

- [ ] 该声明是否回答了某个子问题？
- [ ] 哪个产物支持该声明？
- [ ] 是数值声明吗？→ 数值是否存在于结果文件中？
- [ ] 需要基线对比吗？→ 基线结果是否存在？
- [ ] 需要鲁棒性证据吗？→ 鲁棒性检查是否完成？
- [ ] 措辞是否过强？

**有证据的弱声明 > 无证据的强声明。**

## 七、摘要写作模式 (v5.2 — 来自 math-modeling-competition-workflow)

### 推荐序列（单段式摘要）

1. **问题与目标**: 陈述数据问题和总体目标
2. **方法链**: 具体说明方法链，而非仅列举模块名
3. **逐题结果**: 对每个子问题，使用统一句式
4. **关键数值**: 仅包含正文中出现的关键数值结果
5. **总结**: 可靠性、可解释性和实际意义

### 逐题句式模板

对每个子问题，使用以下句式：

> **"针对问题 X，考虑……，建立……模型，采用……算法，得到……结果。"**

示例：
> "针对问题一，考虑评委评分的主观偏差和尺度差异，建立基于MAD的离群检测与Z-score标准化模型，采用三步分数调整算法，得到消除偏差后的标准评分矩阵，Spearman相关系数从0.67提升至0.89。"

### 方法链写法示例

好的方法链描述应具体说明每步如何衔接：

`
本文首先以滑动残差偏差校正模型统一两类传感器尺度；
随后基于Hampel跳变识别和窗口速度平滑，建立受约束三段回归模型识别阶段；
再采用滚动中位数插补和共同异常指标完成多源数据质量控制；
在此基础上训练分阶段梯度提升模型预测位移；
最后以留一变量组合选择和阶段内速度分位数构建四级预警阈值。
`

### 禁止的写法

- ❌ "本文针对XX问题进行了深入研究" — 空洞
- ❌ "建立了数学模型" — 未说明模型类型
- ❌ "取得了良好的效果" — 未给出具体数值
- ❌ "具有重要的理论意义和实践价值" — 套话
- ❌ 摘要中出现正文未出现的数值 — 违反结果完整性铁律

## 八、求解章节写作模式 (v5.2 — 来自 math-modeling-competition-workflow)

### 每个子问题求解的标准五步结构

| 步骤 | 内容 | 要求 |
|------|------|------|
| **1. 实现来源** | 说明代码文件路径、关键库、可复现路径 | 让读者能重复结果 |
| **2. 求解步骤** | 按顺序描述求解过程 | 每步配文字解释 |
| **3. 图表插入** | 在支撑该结果的段落紧后放置图表 | 图表不出现在无关段落 |
| **4. 三段论分析** | 描述→分析→结论 | 每张图表都强制执行 |
| **5. 直接答案** | 以明确语句回答子问题 | 不加修饰、不绕弯 |

### 求解步骤的写法

`
以问题X的求解为例：

步骤1：[操作描述，如"使用XXX库读入评分矩阵S_{ij}"]
步骤2：[操作描述，如"对每个评委j，计算其评分与总体均值的偏差"]
步骤3：[操作描述，如"对偏差>3倍MAD的评分标记为离群值"]
...
步骤N：[最终输出，如"输出标准评分矩阵Final_{ij}及离群检测报告"]

求解结果如图X和表X所示。
`

### 禁止

- ❌ 只说"使用Python求解"而不给出具体步骤
- ❌ 图表出现在结论段落之后（图表应在支撑段落紧后）
- ❌ 图表只有标题，无正文中的三段论分析
