# Nature-Style 数学建模论文写作指南

> 集成自 `nature-writing` skill，专为数模竞赛论文优化。

---

## 核心原则

1. **先有论据，再有句子。** 不编造结果、机制、参考文献。
2. **每段一个信息。** 段首句声明段落目的，后续句展开。
3. **主张要有边界。** 不用"显著提高"而不给基线，不用"首次"而不限范围。
4. **观察归结果，含义归讨论。** 不把相关性当因果。

---

## 数模论文常见 AI 写作痕迹及修复

| AI 痕迹 | 问题 | 修复 |
|---------|------|------|
| `本文针对...问题，建立了...模型` | 千篇一律的开头 | 先讲清楚"什么技术难题、为什么现有方法不行" |
| `首先...其次...最后...` | 流水账结构 | 改为"挑战→贡献→证据"逻辑链 |
| `显著提高/明显改善` | 无基线对比 | 加上"相比XX方法，nRMSE 从 0.23 降至 0.15" |
| `具有重要的理论意义和工程价值` | 空洞拔高 | 具体说明"可使电网调度误差降低 X%" |
| `首次提出/创新性地` | 无边界声明 | 改为"针对 XX 场景，提出了 YY 方法" |
| `实验结果表明...证明了...` | 因果倒置 | 改为"suggests / is consistent with / 表明...与...一致" |
| 摘要 = 论文目录 | 摘要不是结构列表 | 用"挑战→方法→关键结果→意义"四段式 |

## 各章节写法要点

### 摘要（Abstract）

**结构：** 任务背景 → 技术挑战 → 本文方法（1-2句） → 关键数值结果 → 意义

```
❌ 错误：本文针对XX问题，首先建立了AA模型，其次构建了BB模型，
        最后通过CC验证。结果表明模型有效。

✅ 正确：XX问题的核心难点在于YY。现有方法受限于ZZ，难以同时满足
        AA和BB。本文提出CC方法，通过DD机制解决了EE矛盾。
        在FF数据集上，nRMSE 从 0.23 降至 0.15（降幅 35%），
        为GG场景提供了可行的HH方案。
```

### 问题重述（Introduction）

**倒推法：** 先回答 4 个问题再动笔：
1. 解决什么技术问题？为什么没有现成方案？
2. 本文的技术贡献是什么？
3. 方法为什么能 work？
4. 提供了什么新认知？

### 模型建立（Method）

- 先声明模型类型："建立[线性规划/微分方程/XGBoost集成]模型"
- 再列出完整变量表（符号、中文名、单位、类型）
- 写出完整数学公式（目标函数 + 约束条件）
- 说明假设及其依据

### 结果分析（Results/Experiments）

- 每个图表一个信息点
- 图表标题写结论不写内容（如"XGBoost 在阴天场景下误差降低 40%"而非"不同模型误差对比"）
- 结果段落结构：① 实验设置 → ② 观察 → ③ 对比 → ④ 解释

### 结论（Conclusion）

**三段式：**
1. 贡献总结（做了什么，关键数字）
2. 局限性（诚实列出 2-3 条）
3. 未来方向（基于局限性的自然延伸）

## 段落流畅性检查（Paragraph Flow）

写完每段后自查：

1. 这段只有一个信息吗？
2. 首句声明了段落目的吗？
3. 句与句之间是因果/对比/递进/举例关系吗？
4. 有没有句子属于另一段？

**反向大纲法：** 
- 写下每段主题句 → 对照全文论点 → 删掉无法映射的段落

## 自查清单（提交前）

- [ ] 每个声称有数值支撑
- [ ] 没有"显著/明显/大大"等空洞副词
- [ ] 图表标题是结论而非描述
- [ ] 摘要不重复论文结构
- [ ] 结论诚实列出局限性
- [ ] 参考文献 ≥ 10 条，格式统一
