# 模型鲁棒性与灵敏度分析指南

> 集成自 MathModeling-skills robustness-checker。每个模型求解完成后强制执行。

---

## 核心原则

**模型的结论必须经得起扰动。** 如果一个参数 ±10% 结论就翻转，那这个结论不可靠。

---

## 按模型类型的必做检查

| 模型类型 | 必做检查 |
|---------|---------|
| 优化模型 | 基线对比 + 参数灵敏度 + 约束扰动 + 极端场景 |
| 预测模型 | 基线对比 + 参数灵敏度 + 误差分析 + 数据扰动 + 极端场景 |
| 统计模型 | 基线对比 + 参数灵敏度 + 误差分析 + 数据扰动 |
| 评价模型 | 基线对比 + 权重灵敏度 + 参数灵敏度 |
| 分类模型 | 基线对比 + 参数灵敏度 + 误差分析 + 数据扰动 |
| 仿真模型 | 基线对比 + 参数灵敏度 + 随机稳定性 + 极端场景 |
| 机理模型 | 基线对比 + 参数灵敏度 + 极端场景 |

---

## 八类检查详解

### 1. 基线对比 (Baseline)
**必须做。** 拿最简单的模型（均值预测/常数模型/朴素方法）做对照。
```
如果主模型不比基线好 → 诚实报告，不能隐瞒
如果主模型提升 < 5% → 讨论是否值得增加复杂度
```

### 2. 参数灵敏度 (Parameter Sensitivity)
扰动关键参数 ±5%, ±10%, ±20%，观察输出变化。
```
输出变化 < 10% → 结论稳定
输出变化 10-30% → 需要讨论
输出变化 > 30% → 结论脆弱，必须限定条件
```

### 3. 权重灵敏度 (Weight Sensitivity)
仅适用于评价模型。扰动指标权重，检查排序是否改变。

### 4. 数据扰动 (Data Perturbation)
添加噪声 / 删除样本 / Bootstrap 重采样 / 缺失值策略对比。

### 5. 误差分析 (Error Analysis)
检查残差分布（正态？异方差？）、误差模式（系统性偏差？）、离群值影响。

### 6. 随机稳定性 (Randomness Stability)
至少 10 个不同随机种子，报告均值和标准差。

### 7. 极端场景 (Extreme Scenario)
测试边界条件：最大值、最小值、零值、负值（如果合法）。

### 8. 约束扰动 (Constraint Perturbation)
仅适用于优化。放松/收紧约束，检查可行域和最优解变化。

## 报告模板

每个子问题生成 `robustness/Qx/qx_robustness_report.md`：

```
# Qx 鲁棒性报告

## 支持的结论
| 结论 | 支持证据 | 置信度 |
|------|---------|--------|

## 脆弱的结论
| 结论 | 脆弱原因 | 限定条件 |
|------|---------|---------|

## 结论边界
| 结论 | 成立条件 | 可能失效条件 |
|------|---------|-------------|
```

## 使用方法

```bash
# 生成报告模板
python scripts/verify_solution.py --question Q1 --model-type prediction

# 检查模型推导深度
python scripts/verify_solution.py --depth-check "模型描述文本"
```
